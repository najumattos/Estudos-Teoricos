# MVC X DDD
Enquanto a **Arquitetura** define as fronteiras (as paredes da casa), a **Metodologia de Design** define como os moradores (as regras de negócio) se comportam e se comunicam.

## **arquiteturas mais relevantes**

1. MVC
2. Clean Architecture (Arquitetura Limpa)
3.  Hexagonal Architecture (Ports and Adapters)
4. Microservices (Microsserviços)
5. Event-Driven Architecture (Arquitetura Orientada a Eventos)
6. Serverless Architecture

## **metodologia de design**
1. DDD
2. Data-Driven Design (Design Orientado a Dados)
3. Transaction Script
4. Anemic Domain Model (Modelo Anêmico)
5. Event Storming / Event Sourcing (Focado em Comportamento)


#
Se você olhar para as pastas de um projeto MVC e ler "Controllers, Models, Views", você sabe que é um projeto **Web**. Se você olhar para as pastas de um projeto com Clean Architecture bem feita e ler "RealizarEmprestimo, DevolverLivro, CalcularMulta", você sabe que é um projeto de **Biblioteca**, antes mesmo de saber se é Web ou Mobile.
